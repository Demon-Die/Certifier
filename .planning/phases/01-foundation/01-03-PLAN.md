<task type="auto">
  <name>task 2: Implement signIn callback to auto-create/update Supabase profile</name>
  <files>src/lib/auth-options.ts, src/lib/supabase/server.ts</files>
  <action>
In auth-options.ts signIn callback (or jwt callback), use createServerClient() to upsert profile:

```typescript
async signIn({ user, account, profile }) {
  if (!account?.access_token || !profile?.login) return true;
  const supabase = createServerClient();
  
  // Upsert profile
  const { error } = await supabase.from('profiles').upsert({
    id: user.id, // NextAuth uses UUID for user.id
    github_username: profile.login,
    role: 'contributor', // default, only admins can change
    updated_at: new Date().toISOString()
  }, { onConflict: 'id' });
  
  if (error) console.error('Profile upsert error:', error);
  return true; // Allow sign in even if profile creation fails (log only)
}
```

Note: NextAuth v4 uses user.id as string. Ensure Supabase profiles.id matches. If using email as ID, adjust accordingly. The key is linking GitHub account to Supabase profile.

Also in jwt callback: fetch role from profiles table and add to token:
```typescript
const { data: profile } = await supabase.from('profiles').select('role').eq('id', token.sub).single();
if (profile) token.role = profile.role;
```
